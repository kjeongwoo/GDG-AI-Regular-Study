## 지도 학습의 종류
* 분류: 샘플을 몇개의 클래스 중 하나로 분류하는 것
* 회귀: 임의의 어떤 숫자를 예측하는 문제

## 모델 오차 평가

score = 결정계수(R^2) = 1 - 모델 오차: (실제값 - 예측값) ^ 2 / 평균 오차: (실제값 - 평균값) ^ 2   
 
* score = 1: 모델이 평균 오차보다 우수
* score = 0: 평균 오차와 비슷
* score < 0: 평균 오차보다 큼  

MAE: 오차 절댓값의 평균


numpy.reshape: 2차원 배열로 재구성

## 과적합
* 과대적합: 훈련 세트에만 과하게 맞춰지고 변형된 데이터를 맞출 수 없음    
train점수 >>>>> test점수      

* 과소적합: 훈련 세트 규칙 미발견(학습 부족)     
train점수 < test점수   
KNN의 경우 이웃의 개수(k)를 줄이면 훈련 때 입력한 데이터에 더 민감하게 반응 -> 과소적합 해결
## k-최근접 이웃 알고리즘

### 종류
* 분류: 이웃들 중 다수 클래스를 새로운 샘플로 예측
* 회귀: 이웃들의 평균값 산출

### 한계
학습 데이터에서 크게 멀어진 값을 예측 시 오류 발생. 예를 들면 50cm인 물고기와 1,000,000cm인 물고기의 무게를 동일하게 표시할 수도 있다.

### 한계에 대한 해결책: 선형 회귀(직선 학습)

* LinearRegression 클래스 이용, fit, score, predict 이용
단점: 왼쪽 아래 데이터는 설명 어려움(음수 데이터 발생과 같은 이상 현상 발생 가능)   

다항회귀: 여러 개의 특성을 이용한 회귀   
이차함수의 경우 특성을 x 하나로 보지 말고, x^2와 x, 즉 총 2개의 특성을 가진 것으로 간주한다.   

* PolynomialFeatures 클래스: 최대 차수를 지정한 후 주어진 특성의 차수를 조합하여 식 생성   
include_bias=false(선형회귀 클래스에 이미 포함됨)   
차수 너무 높이면 과대적합(너무 학습 데이터에만 맞춰진 상태) 발생

## 규제(과대적합 방지)   
차수가 과하게 큰 항이 식 전체에 과하게 영향을 끼치는 것을 막기 위해 항의 계수 조절   
차수가 너무 크면 규제만으로는 힘드니 스케일링 필요   
* Ridge: 전체적인 계수의 값 축소   
* Lasso: 일부 계수의 값을 0으로 전환

## 느낀 점
최근 AI에 대해서 학습 데이터의 양이 부족하다는 소식이 많이 들려오고 있는데, 이번 수업 시간을 통해 단순히 데이터를 많이 넣는다고 해결되는 것이 아니라 모델이 데이터를 일반화하여 규칙을 찾을 수 있게 하려면 규제나 선형회귀 등의 방식을 이용하여 입력 데이터를 상황에 맞게 선별해야 한다는 것을 깨달았다.
