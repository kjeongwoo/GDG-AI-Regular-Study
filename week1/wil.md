# AI 정규 스터디 1주차 WIL

## 배운 점
### 머신러닝 VS 딥러닝
* 딥러닝은 머신러닝에 포함
* 인공신경망을 사용해서 연산능력 상승

## 파이썬을 사용하는 이유
* 라이브러리 생태계 좋음
    * 파이썬: 모델 실험 설계
    * 라이브러리: 실제 계산 수행

## 라이브러리 종류

* pandas: 데이터 처리 라이브러리
* numpy: 수치 계산 라이브러리
* matplotlib: 데이터 시각화 라이브러리
* scikit-learn: 머신러닝 라이브러리

## 모델 학습
* 모델을 학습시킨다 = 문제 + 정답 공부  
* 라벨링: 문제의 "정답"을 모델에 알려주기   

* KNN: 데이터와 가장 가까운 이웃 데이터 K개를 보고 그것들의 정답에 따라 다수결로 판단.   
* "정확도 100%" -> 훈련 데이터 = 테스트 데이터란 뜻. 지양할 것(훈련 데이터와 테스트 데이터 나누기)


## 실습 결과   
실험용으로 투입한 세 가지 데이터 중 [30, 600]과 [12, 10]은 시각 데이터와 예측 데이터 모두 일치하지만,   
[25, 150]의 예측 데이터가 시각 데이터와 다르게 도미가 아니라 빙어로 나옴.

## 느낀 점   
데이터를 학습하는 모델을 개발자가 직접 설계하는 것이 아니라 이미 라이브러리에 구현된 기능을 활용한다는 것이 놀라웠다. 또한, 가지고 있는 모든 데이터를 학습에 사용하지 말고 일부는 테스트 데이터용으로 남겨놓는 것이 좋다는 내용을 듣고, "학습용 데이터와 테스트용 데이터를 구분할 때 학습용에 좋은 데이터가 따로 있고 테스트용에 좋은 데이터가 정해져 있을까? 정해져 있다면 그것을 구분하는 기준은 무엇일까?" 라는 생각이 들었다.


# 배운 함수
* zip(배열1, 배열2): 배열 데이터 차례대로 묶어서 반환

* matplotlib
    * scatter: 산점도 데이터 입력
    * xlabel: x축 라벨링
    * ylabel: y축 라벨링
    * show: 그래프 표시

* from sklearn.neighbors import KNeighborsClassifier
    * KNeighborClassifier(): KNN 모델
    * fit: 훈련(2차원 배열 입력)
    * score: 모델 정확도 평가(정답 개수/문제 개수)
    * predict: 문제 풀이
